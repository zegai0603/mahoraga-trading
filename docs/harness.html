The Harness

A Cloudflare Durable Object that runs 24/7 without a local machine.

What is the Harness?

The Harness (MahoragaHarness) is an advanced trading agent that runs as a Cloudflare Durable Object. Unlike the Node agent which requires a local machine, the Harness runs in the cloud with:

  • 24/7 uptime — No local machine required
  • Persistent state — Survives restarts and deployments
  • Crypto trading — Trade BTC, ETH, SOL around the clock
  • Multi-source signals — StockTwits + Reddit + Twitter confirmation
  • Automatic scheduling — Alarm-based execution every 30 seconds

Quick Start

1. Deploy to Cloudflare

npx wrangler deploy

2. Set Secrets

npx wrangler secret put ALPACA_API_KEY
npx wrangler secret put ALPACA_API_SECRET
npx wrangler secret put OPENAI_API_KEY
npx wrangler secret put KILL_SWITCH_SECRET

# Optional
npx wrangler secret put ALPACA_PAPER           # "true" for paper trading
npx wrangler secret put TWITTER_BEARER_TOKEN
npx wrangler secret put DISCORD_WEBHOOK_URL

3. Enable the Agent

curl https://your-worker.workers.dev/agent/enable

4. Monitor

# Check status
curl https://your-worker.workers.dev/agent/status

# View logs
curl https://your-worker.workers.dev/agent/logs

# Check costs
curl https://your-worker.workers.dev/agent/costs

Local Development

# Terminal 1 - Start wrangler
npx wrangler dev

cd dashboard && npm run dev

# Terminal 3 - Enable
curl http://localhost:8787/agent/enable

# Manual trigger (for testing)
curl http://localhost:8787/agent/trigger

Customization Guide

The Harness file (src/durable-objects/mahoraga-harness.ts) is documented with markers to help you find what to modify:

Marker Meaning
[TUNE] Numeric values you can adjust (thresholds, limits, percentages)
[TOGGLE] Features you can enable/disable
[CUSTOMIZABLE] Sections with code you might want to modify

Adding a New Data Source

The most common customization is adding a new data source. Here's how:

1. Create a gather method

private async gatherMySource(): Promise<Signal[]> {
  const signals: Signal[] = [];
  
  try {
    // Fetch from your API
    const res = await fetch("https://your-api.com/data");
    const data = await res.json();
    
    for (const item of data.items) {
      const tickers = extractTickers(item.text);
      const sentiment = detectSentiment(item.text);
      
      for (const symbol of tickers) {
        signals.push({
          symbol,
          source: "my_source",
          source_detail: "my_source_detail",
          sentiment: sentiment * 0.9,  // Apply source weight
          raw_sentiment: sentiment,
          volume: 1,
          freshness: 1.0,
          source_weight: 0.9,
          reason: `MySource: ${item.summary.slice(0, 50)}`,
        });
      }
    }
  } catch (error) {
    this.log("MySource", "error", { message: String(error) });
  }
  
  return signals;
}

2. Add to runDataGatherers

private async runDataGatherers(): Promise<void> {
  const [stocktwitsSignals, redditSignals, cryptoSignals, mySignals] = 
    await Promise.all([
      this.gatherStockTwits(),
      this.gatherReddit(),
      this.gatherCrypto(),
      this.gatherMySource(),  // Add here
    ]);
  
  this.state.signalCache = [
    ...stocktwitsSignals, 
    ...redditSignals, 
    ...cryptoSignals,
    ...mySignals,  // And here
  ];
}

3. Add source weight

const SOURCE_CONFIG = {
  weights: {
    stocktwits: 0.85,
    reddit_wallstreetbets: 0.6,
    // ...
    my_source: 0.9,  // Add your source weight
  },
  // ...
};

Modifying LLM Prompts

The LLM prompts are in the researchSignal() and analyzeSignalsWithLLM() methods. Here's the signal research prompt:

const prompt = `Should we BUY this stock based on social sentiment?

SYMBOL: ${symbol}
SENTIMENT: ${(sentimentScore * 100).toFixed(0)}% bullish
SOURCES: ${sources.join(", ")}

Evaluate if this is a good entry. Consider:
- Is the sentiment justified?
- Is it too late (already pumped)?
- Any red flags?

JSON response:
{
  "verdict": "BUY|SKIP|WAIT",
  "confidence": 0.0-1.0,
  "entry_quality": "excellent|good|fair|poor",
  "reasoning": "brief reason",
  "red_flags": ["any concerns"],
  "catalysts": ["positive factors"]
}`;

Modify the prompt to add your own criteria, context, or change the output format.
